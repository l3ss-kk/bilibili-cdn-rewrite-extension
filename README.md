# 解决部分 B 站视频播放卡顿

对 B 站视频频繁缓冲有奇效，尤其是直播回放、长视频这种一挂就容易卡的场景。

平时经常挂 B 站直播回放还有一些视频播一会就开始卡，后来发现换一条加载线路后会稳定很多，顺手写了个插件。对我很有用，海外用户或者网络到 B 站不太稳定的情况也可以试试。


## 适合什么情况

- 视频能打开，但播放过程中频繁缓冲
- 直播回放、长视频容易播着播着卡住
- 同一个视频有时流畅有时卡
- 海外用户访问 B 站时默认线路不稳定
- 想快速试试换一条加载线路会不会更顺

## 怎么用

1. 在右侧 **Releases** 点最新版本，比如 `v0.6.0` 或带 `Latest` 的那个
2. 在页面里的 **Assets** 里下载 zip 压缩包
3. 把 zip 解压出来，得到一个插件文件夹
4. 打开 Chrome，地址栏输入 `chrome://extensions/`
5. 开启右上角的 **开发者模式**
6. 点击 **加载已解压的扩展程序**
7. 选择刚才解压出来的插件文件夹
8. 打开或刷新 B 站视频页面
9. 点击扩展图标，选择一条线路

选择线路后，如果当前标签页是 B 站页面，插件会尝试自动刷新；如果没有刷新，手动刷新一次视频页面即可。

加载后先不要删除这个文件夹，Chrome 会从这个文件夹里读取插件。

## 可选线路

- `mirrorhw`
- `mirrorali`
- `mirroralib`
- `mirrorhwb`


## 原理 / 当前规则

插件会把部分 B 站视频加载请求改到你选择的线路上。当前处理的来源线路包括：

| 原始线路 | 可切换到 |
| --- | --- |
| `upos-sz-mirrorcosov.bilivideo.com` | `mirrorhw` / `mirrorali` / `mirroralib` / `mirrorhwb` |
| `upos-hz-mirrorakam.akamaized.net` | `mirrorhw` / `mirrorali` / `mirroralib` / `mirrorhwb` |

## 隐私

插件只保存你选择的线路，不上传数据，不读取账号信息。
